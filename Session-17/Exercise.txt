1. Πως μπορούμε να κάνουμε ένα Request σε ένα εξωτερικό API χωρίς να γνωρίζει ολόκληρο το url μας;

-> Οχι, δεν μπορούμε να κάνουμε ένα Request σε ένα εξωτερικό API χωρίς να γνωρίζουμε ολόκληρο το url. 
	Για να κάνουμε ένα Request για ένα resource στο Web απαιτεί την πλήρη διεύθυνση URL(Uniform Resource Locator) αυτού του resource. 
	Μια διεύθυνση URL περιέχει όλες τις απαραίτητες πληροφορίες για ανάκτηση και τον εντοπισμό του resource, 
	συμπεριλαμβανομένου του πρωτοκόλλου (π.χ. HTTP), την διεύθυνση IP ή το όνομα του domain καθώς και το Path του resource στον σερβερ.
	Χωρίς να γνωρίζουμε ολόκληρο το url δεν είναι μπορούμε να κάνουμε Request του αιτήματος σε ένα εξωτερικό API.

2. Αν κάνω ενα GET request και ζητήσω ενα jpeg image, το response τι content-type πρεπει να εχει;

->  Εάν υποβάλλουμε ένα GET request για μια εικόνα .jpeg, ο τύπος περιεχομένου της απάντησης θα είναι συνήθως "image/jpeg". 
Ο τύπος περιεχομένου είναι ένα πεδίο κεφαλίδας στην απόκριση HTTP που καθορίζει τον τύπο περιεχομένου που επιστρέφεται από τον διακομιστή. 
Αυτό γίνεται για τη σωστή λειτουργία ως προς τον χρήστη.

3. Πως μπορώ να κάνω ένα GET request και το περιεχόμενο του να κρατηθεί στην cache για 1 μέρα;

->Για να κάνω ένα αίτημα GET request με διάρκεια προσωρινής μνήμης 1 μέρας, μπορούμε να να βάλουμε ενα header στο request με ενα value που θα είναι ίσο 86400. 
Αυτο το header στην ουσία "ενημερώνει" το περιεχόμενο του να κρατηθεί στην cache για 86400 δευτερόλεπτα (1 μέρα).

4. Μπορώ να έχω μια εφαρμογή που κάνει μόνο GET requests;

->Ναι, είναι δυνατό να έχουμε μια εφαρμογή που κάνει μόνο GET request.Τα GET request είναι ο πιο συνηθισμένος τύπος αιτημάτων HTTP 
και χρησιμοποιούνται για την ανάκτηση πόρων από έναν server.

5. Μπορώ να έχω μια εφαρμογή που κάνει μόνο POST requests;

->Ναι, είναι δυνατό να έχετε μια εφαρμογή που κάνει μόνο αιτήματα POST.
Τα αιτήματα POST χρησιμοποιούνται για την αποστολή δεδομένων σε έναν server για επεξεργασία,
δημιουργία, ενημέρωση και αποστολή resources.

6. Τι status code περιμένω από ένα API αν δεν έχω στείλει ένα
υποχρεωτικό πεδίο;

-> Εάν υποβάλλω ένα αίτημα σε ένα API και δεν έχω στείλει ένα υποχρεωτικό πεδίο, τότε
θα λάβουμε ένα status code "400 Bad Request" ως απόκριση. Ο κωδικός κατάστασης 400 Bad Request υποδεικνύει ότι ο διακομιστής 
δεν μπορούσε να κατανοήσει το αίτημα λόγω μη έγκυρων ή ελλειπόντων απαιτούμενων παραμέτρων.

7. Τι status code περιμένω από ένα API αν δεν υπάρχει επικοινωνία με τη βάση;

-> Εάν υποβάλλω ένα αίτημα σε ένα API και υπάρχει πρόβλημα με τη σύνδεση με τη βάση δεδομένων,
τότε θα λάβουμε ένα status code "500 Internal Server Error" ως απόκριση. Ο κωδικός κατάστασης 
500 Internal Server Error υποδεικνύει ότι ο διακομιστής αντιμετώπισε μια απροσδόκητη συνθήκη 
που τον εμπόδισε να εκπληρώσει το αίτημα.

8. Μπορώ να κάνω Login με GET request;

9. Ποια είναι η διαφορά ανάμεσα στο PUT και στο POST;

10. Ποια μέθοδο θα πρέπει να χρησιμοποιήσω για να «ανεβάσω» ένα αρχείο;
